### 一、发明名称

### 三、技术领域

本发明涉及自动驾驶保险精算（Autonomous Vehicle Insurance Actuarial）技术领域，特别涉及一种融合Robotaxi事故数据库与航测自然驾驶行为基线数据，通过百分位定位算法量化事故可预防性并输出差异化保费的方法及系统。

### 四、背景技术

#### 4.1 现有技术

**（1）Swiss Re与Waymo的合作保险定价**

Swiss Re与Waymo于2024年合作，基于Waymo自有的400万+英里自动驾驶里程数据优化保险定价模型。该方案的核心是将Waymo的事故数据（频率+严重度）与传统车险精算基准对比，输出运营商特定的保费。局限在于：（1）仅使用Waymo自有数据，无法迁移到其他运营商；（2）缺乏"人类驾驶员在相同场景下的表现基线"，无法判断事故是否"本可避免"；（3）定价模型为黑盒，监管难以审核。

**（2）英国AV Act 2024保险责任框架**

英国《自动驾驶车辆法案2024》要求ASDE（授权自动驾驶实体）持有责任保险。但保险金额的计算依赖传统精算方法——基于历史索赔频率×平均赔付额，未引入"可预防性"维度。

**（3）中国平安/人保的智能网联汽车保险产品**

中国平安、人保等保险公司正在探索智能网联汽车保险产品，但目前仍沿用传统定价模型。国内尚无发表的公开方案将"自然驾驶行为基线"作为保险定价的分母。

**（4）学术界的可预防性评估**

Cui等（2023）在交通事故分析领域提出过"Preventability"概念，但仅作为定性描述（preventable/not-preventable），缺乏基于行为数据的量化算法。没有学术工作将百分位分布作为可预防性判据。

#### 4.2 现有技术的不足

1. **缺乏"行为基准"导致定价不精准**：传统保险精算基于频率-严重度矩阵，未区分"该事故是否人类驾驶员也会发生"。本应承担高保费的系统缺陷可能按自然风险定价；本应低保费的不可抗力事件可能按缺陷定价。

2. **事故数据与正常驾驶数据未融合**：现有方案单独分析事故数据（分子），缺乏与自然驾驶行为基线数据（分母）的结合。这导致无法回答保险精算的核心问题："该事故在当前场景的百分位是多少？"

3. **可预防性判定依赖人工专家评审**：传统事故调查依赖专家主观判断事故是否可避免。这种方法成本高、速度慢、一致性差，无法应用于大规模Robotaxi运营。

4. **缺乏差异化运营商评级**：现有方案按运营商整体出险率评级，未反映运营商在"可预防性事故"上的具体表现。两家运营商可能总事故数相同，但一家多为PI-A（不可预防），一家多为PI-C（应预防），风险特性完全不同。

5. **城市环境因子未精细化**：通用保险的城市差异系数基于宏观统计（如理赔金额），未反映Robotaxi特定的场景复杂度。

6. **未对齐中国GB标准的残余风险准则**：中国GB《智能网联汽车 自动驾驶系统安全要求》附录D明确残余风险接受准则（碰撞率<10⁻⁴/h、致命率<10⁻⁷/h），但现有保险精算方法未与这些量化准则对接。

7. **缺乏独有的大规模自然驾驶数据源**：构建可预防性基线需要大规模、高质量、细粒度的自然驾驶行为数据。目前全球范围内缺乏能够支撑中国城市交通场景分析的航测自然驾驶数据集。

#### 4.3 本发明要解决的技术问题

- 问题P1：如何通过自然驾驶行为基线量化每起Robotaxi事故的可预防性？（对应不足1-3）
- 问题P2：如何实现运营商级别的差异化风险评级？（对应不足4）
- 问题P3：如何将定价模型与GB标准残余风险准则量化对接？（对应不足6）
- 问题P4：如何基于航测数据（区别于浮动车/定点摄像头数据）的独有属性实现高精度基线？（对应不足7）

### 五、发明内容

#### 5.1 技术方案概述

本发明提出一种基于事件数据库与自然驾驶基线融合的Robotaxi事故可预防性量化分级与风险定价方法，包括以下步骤：

- **S1**（事件要素标注）：从Robotaxi运营事件数据库中提取每起事故的场景类型、车辆行为参数（速度、TTC、THW、加速度）、环境条件（道路类型、天气、时段）。
- **S2**（自然驾驶基线匹配）：从航测自然驾驶数据集（DRIVEResearch航测NDD，750h+飞行，1050万+轨迹）中按相同场景+环境+速度区间检索匹配样本，计算P5/P25/P50/P75/P95百分位分布。
- **S3**（可预防性指数PI计算）：将事故车辆的行为参数定位到基线分布的百分位位置，根据位置判定PI等级：PI-A（<P5，不可预防）、PI-B（P5-P25，边界可预防）、PI-C（>P25，应可预防）。
- **S4**（事故暴露率计算与GB标准对齐）：计算每运营商的各类事故率（按小时），与GB标准附录D残余风险准则量化对比。
- **S5**（差异化保费定价）：最终保费=基础保费×PI系数×运营商评级系数×城市环境系数，支持按运营商、城市、场景三维差异化。

#### 5.2 步骤S3：可预防性指数PI计算（核心创造性）

**（a）PI三级定义**

```
PI-A（不可预防）: 事故行为位于基线P5以下 → 人类驾驶员也无法避免 → 保费系数C_A=1.0
PI-B（边界可预防）: 事故行为位于P5-P25之间 → 少数优秀人类驾驶员可避免 → C_B=1.5
PI-C（应可预防）: 事故行为位于P25以上 → 大多数人类驾驶员可避免 → C_C=3.0
```

**（b）PI计算算法**

```python
def calculate_pi(incident_vector, baseline):
    # 提取事故车辆的关键行为参数
    behavior = extract_behavior(incident_vector)

    # 定位百分位
    percentile = locate_percentile(behavior, baseline.percentiles)

    # PI判定
    if percentile < 5:
        return PI_A  # 不可预防
    elif percentile < 25:
        return PI_B  # 边界可预防
    else:
        return PI_C  # 应可预防
```

**（c）关键参数选择**

不同场景类型使用不同的关键行为参数：

| 场景类型 | 关键参数 |
|---------|---------|
| C1 路中停车 | 停车位置的TTC+横向偏移 |
| B4 行人冲突 | 制动距离+反应时间 |
| C5 低速行人碰撞 | 碰撞时速度+制动决策时机 |
| E3 应急车辆冲突 | 让行时机+横向位置 |

每个场景的关键参数由该场景的物理安全机理决定，不可通用替换。

**反"换应用场景"测试：** PI-A/B/C的阈值（P5、P25）由"人类驾驶员行为分布的统计学常识"决定——P5代表"极少数情况下的极端行为"，P25代表"1/4驾驶员的典型行为"。这一判据不可简单迁移到其他领域（如医疗诊断的置信度阈值通常是P95而非P5），因为其语义基于交通安全领域的特有约定。

#### 5.3 步骤S2：百分位定位算法（核心创造性）

**（a）航测NDD数据独特性**

本方案依赖的自然驾驶基线来自驭研科技航测数据集，具有以下独特性：
- **航测视角**：上帝视角观测，无遮挡，同时记录多车交互
- **数据规模**：750h+飞行，1050万+轨迹
- **细粒度**：25Hz频率，可提取TTC、THW、加速度等微观参数
- **覆盖度**：10+城市、多种道路类型、多季节

相比浮动车GPS数据（10秒1次，缺乏交互）和定点摄像头数据（视角受限），航测数据是唯一能提供"多车交互场景的微观行为分布"的数据源。

**（b）场景匹配算法**

```python
def match_baseline(incident, ndd_database):
    filters = {
        'scenario_code': incident.scenario_code,
        'road_type': incident.road_type,
        'speed_range': speed_to_range(incident.speed_estimate_kmh),
    }
    # 从NDD数据库检索匹配样本
    samples = ndd_database.query(filters)

    # 至少需要100条匹配样本才有统计意义
    if len(samples) < 100:
        return None  # 置信度不足

    return Distribution(
        ttc_percentiles=compute_percentiles(samples['ttc']),
        thw_percentiles=compute_percentiles(samples['thw']),
        ...
    )
```

**（c）插值定位**

基线分布给出5个百分位点(P5, P25, P50, P75, P95)，事故行为的具体百分位通过线性插值计算：

```python
def locate_percentile(value, percentiles):
    p5, p25, p50, p75, p95 = percentiles

    if value <= p5:
        return 5 * (value / p5)  # <P5区间
    elif value <= p25:
        return 5 + 20 * (value - p5) / (p25 - p5)
    elif value <= p50:
        return 25 + 25 * (value - p25) / (p50 - p25)
    elif value <= p75:
        return 50 + 25 * (value - p50) / (p75 - p50)
    elif value <= p95:
        return 75 + 20 * (value - p75) / (p95 - p75)
    else:
        return min(95 + 5 * (value - p95) / p95, 100)
```

**（d）置信度调整**

PI判定结果附带置信度：
- 基线样本≥1000：confidence=0.9
- 基线样本100-1000：confidence=0.7
- 基线样本<100：confidence=0.4（视为不可用，需补充数据）

#### 5.4 步骤S4：GB标准残余风险对齐

**（a）GB标准准则**

中国GB《智能网联汽车 自动驾驶系统安全要求》附录D：
- 碰撞事故率 < 10⁻⁴/h
- 轻伤事故率 < 10⁻⁵/h
- 重伤事故率 < 10⁻⁶/h
- 致命事故率 < 10⁻⁷/h

**（b）合规性计算**

```python
def compliance_check(operator_incidents, operating_hours, gb_thresholds):
    rates = {
        'collision': count_collisions(operator_incidents) / operating_hours,
        'fatal': count_fatalities(operator_incidents) / operating_hours,
        ...
    }
    return {
        cat: rates[cat] < threshold
        for cat, threshold in gb_thresholds.items()
    }
```

**（c）合规性→定价映射**

```
GB完全合规 → 保费无加成
单项不合规 → 保费加成20%
多项不合规 → 保费加成50%+监管报告
严重不合规（超出2倍） → 保费加成100%+暂停承保
```

#### 5.5 步骤S5：差异化定价模型

**（a）最终保费公式**

```
Premium = Base_Premium × C_PI × C_Operator × C_City
```

**（b）PI加权系数**

```
C_PI = (Σ N_PI_k × C_k) / Σ N_PI_k

其中：
k ∈ {A, B, C}
N_PI_k = 运营商PI-k类事故数
C_k = PI-k的保费系数（1.0, 1.5, 3.0）
```

**（c）运营商评级系数**

```
PI-C占比 | 评级 | C_Operator
<5%      | ★★★★★ | 0.9
5-10%    | ★★★★  | 1.0
10-20%   | ★★★   | 1.2
20-40%   | ★★    | 1.5
>40%     | ★     | 2.0
```

**（d）城市环境系数**

基于城市场景复杂度（道路密度、交通多样性、行人密度）：
- 一线复杂城市（北京/上海/深圳/广州）：1.2
- 二线常规城市（武汉/成都/杭州）：1.0
- 郊区简单环境：0.8

**权利要求7（系统）：** 一种执行权利要求1-6任一所述方法的Robotaxi风险定价系统，其特征在于，包括：事件数据解析模块、自然驾驶基线匹配模块、百分位定位模块、PI计算模块、GB标准合规性检查模块、差异化定价模块。

- "分子分母"融合（事故数据+基线数据）的思想源于流行病学暴露率方法，但将其应用于保险定价是本领域的创新。
- 结论：通过测试。

**3. 技术效果非显而易见性测试：**

- 对比英国AV Act的通用保险：本方法与GB标准残余风险准则直接对接。
- 结论：通过测试。

**4. 实施例真实可行性测试：**
